# 소울 코어 스왑 시스템 설계

## 1. 범위

### 포함
- 소울 데이터 구조 (PlayerSoulController)
- 스왑 파이프라인 (스탯/체력/스킨/무기/스킬 교체)
- 스왑 가드 조건
- 입력 액션 추가
- 테스트 환경 (하드코딩 2소울 지급)

### 제외
- 스위칭 효과 (별도 문서)
- 서브 패시브 (별도 문서)
- 드랍/획득 플로우
- UI/팝업
- "2슬롯 꽉 찼을 때" UX

---

## 2. 아키텍처

```
PlayerCharacter (파사드 - 외부 진입점)
├─ PlayerMovementController (MonoBehaviour - 이동/물리)
├─ BSC (plain class - 전투)
│  ├─ SSC (스킬)
│  └─ WSC (무기/기본공격)
├─ SpineController (MonoBehaviour - 애니메이션)
└─ PlayerSoulController (plain class - 소울 데이터/가드)
```

- **PlayerCharacter.SwapSoul()** 이 오케스트레이터 역할
- **PSC**는 소울 데이터 관리 + 가드 체크만 담당
- 각 시스템(WSC/SSC/SpineController/AttributeSet)은 자기 영역 교체만 수행

### PlayerSoulController (PSC) 위치/속성
- **파일 위치**: `Assets/02. Scripts/FieldObject/FieldCharacter/Player/PlayerSoulController.cs`
- **타입**: plain class (MonoBehaviour 아님)
- **소유자**: PlayerCharacter가 생성 및 보유

---

## 3. 스왑 파이프라인

```
PlayerCharacter.SwapSoul()
  if (!PSC.CanSwap()) return            // 가드 체크
  SSC.CancelAllSkills()                 // 진행중 스킬 캔슬
  PSC.SaveCurrentHealth()               // 현재 체력 -> SoulGameData
  PSC.SwapSlotKeys()                    // 메인<->서브 키 교환
  AttributeSet.SwapBaseValues(new)      // baseValue 11종 교체
  PSC.RestoreHealth()                   // lastHealth 복원
  WSC.ChangeWeaponType(newType)         // 무기 교체
  SSC.ChangeSkills(newSkillKeys)        // 스킬 교체 (TBD: 2개 vs 4개)
  SpineController.SetCombinedSkin(newSkinKeys)
  OnSoulSwapped?.Invoke()               // 이벤트 발송
```

## 6. 소울 죽음 / 체력 규칙

- 메인 소울 HP 0 -> **게임오버** (서브로 자동교체 없음)
- 서브 소울 자연회복 **없음** (회복 수단은 극히 제한적으로 별도 시스템)
- 소울별 체력 완전 독립: A소울 100/200, B소울 50/70이면 스위칭 시 체력이 50/70으로 변경
- 체력 클램프: `Min(lastHealth, curMaxHealth)`

---

## 7. 액션 캔슬 규칙 (기획 변경)

기존 "공격/콤보 진행 중 스위칭 불가" 규칙이 폐기됨.

- 기본공격 중 -> 이동만 불가. 점프/대시/스킬/소울교체 모두 즉시 캔슬 허용
- 스킬 캐스팅 중 -> 이동/점프/대시/공격/소울교체 모두로 캔슬 허용
- 대시 중에만 다른 동작 불가 (기존 유지)
