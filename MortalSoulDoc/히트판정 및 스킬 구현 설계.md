# 히트판정 및 스킬 구현 설계

## 히트판정 방식: 혼용 (Overlap + Collider+Trigger)

상황에 따라 두 가지 방식을 혼용한다.

| 판정 유형 | 방식 | 예시 |
|-----------|------|------|
| 즉발 근접 | Physics2D Overlap | 검/대검/단도 기본공격, 라이트닝 볼트 등 즉발 스킬 |
| 이동 투사체 | Collider2D + Trigger | 화살(활 기본공격), 파이어볼 등 투사체 스킬 |
| 지속 장판 | Collider2D + Trigger | 일정 시간 유지되며 틱 데미지를 주는 영역 스킬 |

**판단 기준**: 월드에 일정 시간 존재하는 오브젝트면 Collider+Trigger, 한 순간 판정이면 Overlap.

## 즉발형 히트판정 (Overlap)

기본공격(근접 무기) 및 즉발형 스킬에서 사용.

**기본공격 흐름** (WSC):
1. Spine `hit` 이벤트 발생
2. WeaponSettingData의 HitRange/HitOffset 기반으로 `Physics2D.OverlapBox` 호출
3. 범위 내 적에게 DamageInfo 생성 후 `BSC.TakeDamage` 호출

**즉발형 스킬 흐름** (예: 라이트닝 볼트):
1. 스킬 클래스에서 타겟 위치 탐색
2. 해당 위치에 이펙트(VFX) 재생 (시각 연출용 오브젝트만 스폰)
3. 데이터에 정의된 딜레이 후 `Physics2D.OverlapCircle` 등으로 판정
4. 범위 내 적에게 `BSC.TakeDamage` 호출

BattleObject를 스폰하지 않음. 콜라이더가 필요 없는 판정.

## BattleObject 계층 (Collider+Trigger)

레퍼런스의 SkillObject를 기반으로 네이밍 변경 및 2D 전환.

```
BattleObject (abstract)
├─ ProjectileObject   ← 이동 투사체 (활 기본공격, 파이어볼 등)
└─ AreaObject         ← 지속 장판 (틱 데미지 영역 스킬)
```

WSC(기본공격)와 SSC(스킬) 모두 BattleObject를 스폰할 수 있으므로 "SkillObject"가 아닌 "BattleObject"로 명명.

### BattleObject 책임 범위

판정 + DamageInfo 생성 + `BSC.TakeDamage` 호출까지 담당.
스킬 클래스는 BattleObject를 스폰하고 설정(방향, 속도, 콜백 등)만 넘긴다.
투사체가 적에 부딪히는 시점에 스킬 클래스는 이미 실행이 끝난 상태일 수 있으므로 BattleObject가 자체적으로 데미지 전달까지 처리.

### 타겟 접근 방식

BSC는 MonoBehaviour가 아닌 일반 클래스이므로 충돌체에서 직접 `TryGetComponent<BSC>`가 불가능.

```
TryGetComponent<FieldCharacter>() → .BSC → .TakeDamage(damageInfo)
```

### 레퍼런스 대비 변경사항

| 항목 | 레퍼런스 (3D) | 현재 프로젝트 (2D) |
|------|-------------|-------------------|
| 클래스명 | SkillObject | BattleObject |
| 충돌 콜백 | OnTriggerEnter(Collider) | OnTriggerEnter2D(Collider2D) |
| 유효성 검사 | TryGetComponent\<SSC\> | TryGetComponent\<FieldCharacter\>().BSC |
| 투사체 이동 | Vector3 + LookAt | Vector2 + 2D 회전 |
| 네임스페이스 | MS.Field | MS.Battle (BSC 체계에 맞춤) |

### 레퍼런스에서 유지하는 구조

- `onHitCallback` 패턴: BattleObject는 판정, 데미지 계산은 콜백으로 위임
- `maxAttackCount` / `hitCountPerAttack`: 관통 횟수, 다단히트
- `traceTarget`: 유도 투사체 / 대상 추적 장판
- `duration` + 수명 관리
- AreaObject의 `attackInterval` 기반 틱 데미지
- AreaObject의 OnUpdate에서 죽은 타겟 정리 (OnTriggerExit2D로는 비활성화된 콜라이더 감지 불가하므로)

## 기본공격 무기별 히트판정

| 무기 타입 | 히트판정 방식 |
|-----------|-------------|
| GreatSword (대검) | Overlap |
| OneHandSword (한손검) | Overlap |
| Dagger (단도) | Overlap |
| Bow (활) | ProjectileObject 스폰 |
| Staff (지팡이) | ProjectileObject 스폰 또는 Overlap (구현 시 결정) |
