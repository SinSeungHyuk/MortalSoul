### 작업 요청

1. **기본공격 시스템을 "무기 타입당 1개 클래스"로 리팩터**한다. `BaseWeaponAttack` 추상 클래스를 베이스로 두고, 5종 무기(`GreatSword/OneHandSword/Dagger/Bow/Staff`)마다 전용 서브클래스를 생성한다. 패턴은 `BaseSkill` 상속 구조와 대칭.
2. **기본공격은 데이터 드리븐에서 코드 드리븐으로 전환**한다. 기본공격은 "무기 타입당 정확히 1개"라는 closed-set이므로 JSON 파라미터화의 이득이 없고 족쇄가 된다. `WeaponSettingData` 계열 인프라는 전부 제거한다. 각 무기의 수치/애니키는 해당 클래스 내부에 리터럴로 직접 기재한다.
3. **WSC는 "공격 로직 슬롯"으로 슬림화**한다. 현재 공격 루프를 직접 소유하던 구조에서, `BaseWeaponAttack` 인스턴스를 보유하고 입력을 위임하는 역할로 축소한다. 핸들러 생성은 **리플렉션 팩토리**(`MS.Battle.{WeaponType}Attack`)로 `SSC.GiveSkill` 패턴과 동일하게.
4. **스코프: 아키텍처 + OneHandSword 실제 구현 + 나머지 4종 스텁**. GreatSword/Dagger/Bow/Staff는 클래스 파일만 만들어두고 실제 공격 로직은 비워둔다(`Debug.LogWarning` 수준). 리플렉션 팩토리는 5종 전부 인식.
5. **넉백은 전역 상수화**. 기본공격의 넉백 수치는 모두 동일하다는 규칙을 확정하고 `Settings.BasicAttackKnockback`으로 이동. `AttackComboData.Knockback` 필드는 `WeaponSettingData` 자체가 삭제되므로 자동 소멸.
6. **공격속도 스탯 반영**은 `SpineController.PlayAnimation`에 옵셔널 `_timeScale` 파라미터를 추가하여 처리. 기본공격은 전 흐름이 Spine 이벤트에 gated 되어 있으므로, 공격 애니 재생 시점에 트랙 TimeScale만 넘겨주면 콤보 주기 전체가 자동으로 비례 단축된다.
7. **캔슬 메커니즘은 TCS 자동 캔슬만 사용**. 명시적 `CancellationToken`은 도입하지 않는다. 모든 캔슬 경로(대시/점프/스킬/소울스위칭)가 **새 애니메이션 재생을 동반**하므로 `SpineController.PlayAnimation`이 터뜨리는 `OperationCanceledException`이 공격을 정리한다. 이 설계 철학을 `CLAUDE.md`에도 한 문단 추가한다(스킬은 명시적 토큰, 기본공격은 TCS-only로 **의도적 비대칭**).
8. **BSC 초기화는 단일 시그니처 유지**. 오버로드 분리하지 않고 옵셔널 `EWeaponType?` 파라미터로 플레이어/몬스터를 하나의 `InitBSC`가 모두 처리한다.

#### 3. `BaseWeaponAttack` — 얇은 베이스

**경로**: `Assets/02. Scripts/Battle/WeaponAttack/BaseWeaponAttack.cs`
**네임스페이스**: `MS.Battle`

```csharp
using System;
using Cysharp.Threading.Tasks;
using MS.Field;

namespace MS.Battle
{
    public abstract class BaseWeaponAttack
    {
        public event Action OnAttackStarted;
        public event Action OnAttackEnded;

        public bool IsAttacking => isAttacking;

        protected FieldCharacter owner;
        protected PlayerAttributeSet attrSet;
        protected SpineController spine;
        protected bool isAttacking;

        public virtual void InitWeaponAttack(FieldCharacter _owner, PlayerAttributeSet _attrSet)
        {
            owner = _owner;
            attrSet = _attrSet;
            spine = _owner.SpineController;
        }

        public abstract void OnAttackInput();

        // C# event는 파생 클래스에서 직접 invoke 불가 → 얇은 프록시만 제공
        protected void InvokeAttackStarted() => OnAttackStarted?.Invoke();
        protected void InvokeAttackEnded() => OnAttackEnded?.Invoke();
    }
}
```

**설계 원칙**
- 베이스는 "공통 필드 + 이벤트 + 입력 진입점"만 가진다. 콤보 루프/히트 판정 헬퍼는 **의도적으로 넣지 않는다** — 각 무기 파일이 자기 로직을 통째로 소유해 "한 파일 = 한 무기의 전부"라는 가독성을 우선한다.
- `owner/attrSet/spine`은 Init 시점에 한 번만 캐시.
- `attrSet`은 `PlayerAttributeSet`으로 타입 고정. 기본공격은 플레이어 전용.
- `OnAttackInput`은 `abstract` — 예약(선입력) 처리 방식이 무기마다 다를 수 있으므로 각 서브클래스가 책임.
- `isAttacking`은 base에 남김 — WSC/외부에서 `IsAttacking` 조회 필요.
- `comboIndex`, `isReserveNext` 같은 콤보 전용 상태는 **base에 두지 않는다** — Bow/Staff처럼 콤보 개념이 없는 무기가 있으므로 각 서브클래스가 자기 필드로 선언.

#### 4. `OneHandSwordAttack` — 실제 구현

**경로**: `Assets/02. Scripts/Battle/WeaponAttack/OneHandSwordAttack.cs`

**동작 명세**
- 콤보 2개(`Attack_OneHand1` → `Attack_OneHand2`) 루프
- 각 콤보는 단타(`attack` 이벤트 1회)
- 데미지 배율: 콤보1 = 1.0x, 콤보2 = 1.5x (기존 JSON 의도 유지)
- 히트 판정: `Physics2D.OverlapCircleAll` + `Settings.MonsterLayer`
  - 콤보1: range 1.5, offset 1.2
  - 콤보2: range 1.8, offset 1.4
- 넉백: 전 콤보 `Settings.BasicAttackKnockback` 공통
- 속성: Void(무속성) 고정
- 공격 중 추가 입력 → `isReserveNext = true`로 예약 (중복 입력 무시)
- `combo_ready` 이벤트 들어오면 예약 확인 → 예약 있으면 다음 콤보로 전환, 없으면 `Complete` 대기 후 종료
- 공격속도 반영: 각 콤보 애니 재생 시 `PlayAnimation`의 `_timeScale` 파라미터로 `attrSet.AttackSpeed.Value / 100f` 전달

**수치/애니키 리터럴 인라인 원칙**
- `const` 멤버 변수로 빼지 않는다. 각 콤보별 수치가 쓰이는 위치에 직접 리터럴로 기재한다. 사용처가 한 곳뿐이기 때문에 멤버 변수로 두는 게 오히려 추적 비용을 늘린다.

**facing 처리**
- `SpineController`에 `IsFacingRight` 공개 getter 추가(`skeletonAnimation.Skeleton.ScaleX < 0f`). 히트 판정 중심 위치 계산에 사용.

**히트 처리 내부 흐름**
1. `float facingSign = spine.IsFacingRight ? 1f : -1f;`
2. `Vector2 center = (Vector2)owner.transform.position + new Vector2(offset * facingSign, 0f);`
3. `Physics2D.OverlapCircleAll(center, range, Settings.MonsterLayer)`로 근처 몬스터 수집
4. 각 `Collider2D`에서 `GetComponent<FieldCharacter>()`로 타겟 참조 획득
5. 각 몬스터별 `DamageInfo` 생성
   - Attacker: owner
   - Target: monster
   - AttributeType: Void
   - IsCritic: `Random.value * 100 < attrSet.CriticChance.Value` 굴림
   - Damage: 배율 × `attrSet.BaseAttackPower.Value`, 크리티컬 시 추가로 × `CriticMultiple.Value / 100f`
   - KnockbackForce: `Settings.BasicAttackKnockback`
6. `monster.BSC.TakeDamage(damageInfo)` 호출 (`BSC.TakeDamage` 본체는 현재 TODO — 호출 사이트만 제공)

#### 6. `WeaponSystemComponent` 리팩터

**경로**: `Assets/02. Scripts/Battle/WeaponSystemComponent.cs` (기존 파일 덮어쓰기)

```csharp
using System;
using MS.Data;
using MS.Field;
using UnityEngine;

namespace MS.Battle
{
    public class WeaponSystemComponent
    {
        public event Action OnAttackStarted;
        public event Action OnAttackEnded;

        public EWeaponType CurWeaponType => curWeaponType;
        public bool IsAttacking => curAttack?.IsAttacking ?? false;

        private FieldCharacter owner;
        private PlayerAttributeSet attrSet;
        private EWeaponType curWeaponType;
        private BaseWeaponAttack curAttack;

        public void InitWSC(FieldCharacter _owner, PlayerAttributeSet _attrSet)
        {
            owner = _owner;
            attrSet = _attrSet;
        }

        public void ChangeWeaponType(EWeaponType _weaponType)
        {
            if (curAttack != null)
            {
                curAttack.OnAttackStarted -= OnAttackStartedCallback;
                curAttack.OnAttackEnded -= OnAttackEndedCallback;
            }

            curWeaponType = _weaponType;

            string typeName = $"MS.Battle.{_weaponType}Attack";
            Type attackType = Type.GetType(typeName);
            if (attackType == null)
            {
                Debug.LogError($"[WSC] 무기 공격 클래스를 찾을 수 없음: {typeName}");
                curAttack = null;
                return;
            }

            curAttack = Activator.CreateInstance(attackType) as BaseWeaponAttack;
            if (curAttack == null)
            {
                Debug.LogError($"[WSC] 무기 공격 인스턴스 생성 실패: {typeName}");
                return;
            }

            curAttack.InitWeaponAttack(owner, attrSet);
            curAttack.OnAttackStarted += OnAttackStartedCallback;
            curAttack.OnAttackEnded += OnAttackEndedCallback;
        }

        public void ActivateAttack() => curAttack?.OnAttackInput();

        private void OnAttackStartedCallback() => OnAttackStarted?.Invoke();
        private void OnAttackEndedCallback() => OnAttackEnded?.Invoke();
    }
}
```

**변경 포인트**
- `InitWSC`가 `PlayerAttributeSet`을 인자로 받음 (기존 `_owner`만 받던 구조 확장)
- `ChangeWeaponType`이 `WeaponSettingData` 로드 대신 **리플렉션으로 공격 핸들러 생성**
- `comboIndex`, `curWeaponData`, `isReserveNextCombo`, `isAttacking`, `ActivateAttackAsync` 등 **전부 삭제** → `BaseWeaponAttack`으로 이관
- 이벤트는 핸들러의 이벤트를 포워딩하는 형태 (구독자는 여전히 WSC 이벤트 구독)

#### 7. `BattleSystemComponent.InitBSC` — 단일 시그니처 + 옵셔널 무기 타입

**경로**: `Assets/02. Scripts/Battle/BattleSystemComponent.cs`

오버로드 분리 대신 옵셔널 파라미터로 플레이어/몬스터를 한 메서드가 처리한다. BSC는 플레이어/몬스터 공통 컴포넌트이므로 Init도 하나로 통일하는 게 자연스럽다.

```csharp
public void InitBSC(FieldCharacter _owner, BaseAttributeSet _attrSet, EWeaponType? _weaponType = null)
{
    AttributeSet = _attrSet;
    statusEffectDict = new Dictionary<string, StatusEffect>();

    SSC = new SkillSystemComponent();
    SSC.InitSSC(_owner, _attrSet);

    if (_weaponType.HasValue && _attrSet is PlayerAttributeSet playerAttrSet)
    {
        WSC = new WeaponSystemComponent();
        WSC.InitWSC(_owner, playerAttrSet);
        WSC.ChangeWeaponType(_weaponType.Value);
    }
    else
    {
        WSC = null;
    }
}
```

**호출부**
- 플레이어: `BSC.InitBSC(owner, playerAttrSet, EWeaponType.OneHandSword)`
- 몬스터: `BSC.InitBSC(owner, monsterAttrSet)` — 3번째 파라미터 생략

기존 호출부(플레이어/몬스터 매니저)는 시그니처에 `EWeaponType?` 파라미터만 추가되는 수준의 수정. 몬스터 호출부는 변경 불필요(파라미터 생략됨).

#### 8. `SpineController` — `IsFacingRight` 프로퍼티 + `PlayAnimation` 타임스케일 파라미터

**경로**: `Assets/02. Scripts/FieldObject/FieldCharacter/SpineController.cs`

**(a) `IsFacingRight` 공개 getter 추가**

```csharp
public bool IsFacingRight => skeletonAnimation.Skeleton.ScaleX < 0f;
```

`SetScaleX(true)`가 ScaleX를 -1로 뒤집는 기존 컨벤션에 맞춰 `< 0f` 체크.

**(b) `PlayAnimation`에 `_timeScale` 옵셔널 파라미터 추가** — 별도 헬퍼 메서드를 만들지 않고 기존 함수를 확장한다.

```csharp
public void PlayAnimation(string _animationName, bool _loop, float _timeScale = 1f)
{
    CancelAllWaitTcs();
    var entry = skeletonAnimation.AnimationState.SetAnimation(Settings.SpineMainTrack, _animationName, _loop);
    if (entry != null) entry.TimeScale = _timeScale;
}
```

- 기본값 `1f`라 기존 호출부(Idle/Run/Jump 등)는 동작 변경 없음.
- 공격 호출부는 `spine.PlayAnimation("Attack_OneHand1", false, attrSet.AttackSpeed.Value / 100f)`처럼 한 줄로 공격속도 반영.
- `SetAnimation`은 `TrackEntry`를 반환하므로 생성된 엔트리의 `TimeScale`만 덮어쓰면 끝. 다음 애니메이션이 재생되면 새 엔트리가 생성되고 기본값 1f로 복귀.

#### 11. `CLAUDE.md` 업데이트

세 곳을 수정한다.

**(a) 핵심 스크립트 표의 `WeaponSystemComponent.cs` 설명** — 데이터 드리븐 문구 제거하고 클래스 기반 구조로 개정:

> `WeaponSystemComponent.cs` | 기본공격 전담(플레이어 전용). 무기 타입별 `BaseWeaponAttack` 핸들러를 리플렉션으로 생성/보유하고 입력을 위임. `ChangeWeaponType`으로 핸들러 교체, `ActivateAttack`으로 입력 포워딩.

**(b) "기본공격 콤보 시스템" 섹션에 설계 철학 한 문단 추가**:

> **캔슬 메커니즘의 의도적 비대칭**: 스킬(SSC/BaseSkill)은 명시적 `CancellationToken`을 사용하지만, 기본공격(WSC/BaseWeaponAttack)은 `SpineController`의 TCS 자동 캔슬만 사용한다. 이유: 기본공격은 실행 흐름이 순수하게 Spine 이벤트에만 의존하고(비애니 `await` 없음) 모든 캔슬 경로가 새 애니메이션 재생을 동반하기 때문에 토큰이 불필요하다. 스킬은 `UniTask.Delay`, 외부 조건 대기, 침묵 디버프 등 비애니 캔슬 시나리오가 존재하여 토큰이 필요하다.

**(c) `WeaponSettingData` 관련 기술 제거** — Data 섹션의 SettingData 표에서 `WeaponSettingData.cs` 행 삭제. 관련 언급(데이터 드리븐 기본공격 등)도 정리.

#### Task 4: `OneHandSwordAttack` 실제 구현

**파일**: `Assets/02. Scripts/Battle/WeaponAttack/OneHandSwordAttack.cs` **신규 생성**

```csharp
using System;
using Cysharp.Threading.Tasks;
using MS.Field;
using MS.Utils;
using UnityEngine;

namespace MS.Battle
{
    public class OneHandSwordAttack : BaseWeaponAttack
    {
        private int comboIndex;
        private bool isReserveNext;

        public override void OnAttackInput()
        {
            if (isAttacking)
            {
                isReserveNext = true;
                return;
            }
            ActivateAsync().Forget();
        }

        private async UniTaskVoid ActivateAsync()
        {
            isAttacking = true;
            comboIndex = 0;
            InvokeAttackStarted();

            try
            {
                while (true)
                {
                    float timeScale = attrSet.AttackSpeed.Value / 100f;

                    if (comboIndex == 0)
                    {
                        spine.PlayAnimation("Attack_OneHand1", false, timeScale);
                        await spine.WaitForAnimEventAsync(Settings.SpineEventAttack);
                        DoHit(_range: 1.5f, _offset: 1.2f, _damageMul: 1.0f);
                    }
                    else
                    {
                        spine.PlayAnimation("Attack_OneHand2", false, timeScale);
                        await spine.WaitForAnimEventAsync(Settings.SpineEventAttack);
                        DoHit(_range: 1.8f, _offset: 1.4f, _damageMul: 1.5f);
                    }

                    int finishIdx = await UniTask.WhenAny(
                        spine.WaitForAnimEventAsync(Settings.SpineEventComboReady),
                        spine.WaitForAnimCompleteAsync()
                    );
                    bool comboReady = (finishIdx == 0);

                    if (isReserveNext)
                    {
                        isReserveNext = false;
                        comboIndex = (comboIndex + 1) % 2;
                        continue;
                    }

                    if (!comboReady) break;

                    await spine.WaitForAnimCompleteAsync();
                    break;
                }
            }
            catch (OperationCanceledException) { }
            finally
            {
                comboIndex = 0;
                isReserveNext = false;
                isAttacking = false;
                InvokeAttackEnded();
            }
        }

        private void DoHit(float _range, float _offset, float _damageMul)
        {
            float facingSign = spine.IsFacingRight ? 1f : -1f;
            Vector2 center = (Vector2)owner.transform.position + new Vector2(_offset * facingSign, 0f);
            var colliders = Physics2D.OverlapCircleAll(center, _range, Settings.MonsterLayer);

            foreach (var col in colliders)
            {
                var target = col.GetComponent<FieldCharacter>();
                if (target == null) continue;

                bool isCritic = UnityEngine.Random.value * 100f < attrSet.CriticChance.Value;
                float damage = _damageMul * attrSet.BaseAttackPower.Value;
                if (isCritic) damage *= attrSet.CriticMultiple.Value / 100f;

                var info = new DamageInfo
                {
                    Attacker = owner,
                    Target = target,
                    AttributeType = EDamageAttributeType.None,
                    Damage = damage,
                    IsCritic = isCritic,
                    KnockbackForce = Settings.BasicAttackKnockback,
                    SourceSkill = null
                };

                target.BSC.TakeDamage(info);
            }
        }
    }
}
```

**중요 포인트**
- `DamageInfo`는 **struct + public fields 구조**이므로 **object initializer 구문**(`new DamageInfo { ... }`)으로 생성. 생성자 호출 아님.
- 속성은 `EDamageAttributeType.None` (기존 enum에 `Void`는 없음 — `None = 0`이 무속성 역할).
- 타겟 참조는 `col.GetComponent<FieldCharacter>()`. 몬스터 콜라이더가 자식 오브젝트에 붙어있으면 null 반환 — 런타임 검증에서 문제 발견 시 `GetComponentInParent`로 교체.
- 매개변수명은 프로젝트 규칙대로 `_` 접두사 (named args 사용 시 `_range:`, `_offset:`, `_damageMul:`).

**검증**: 컴파일 에러 없음.

##### 6-2. `WeaponSystemComponent` 리팩터

**파일**: `Assets/02. Scripts/Battle/WeaponSystemComponent.cs` **기존 파일 전체 덮어쓰기**

```csharp
using System;
using MS.Data;
using MS.Field;
using UnityEngine;

namespace MS.Battle
{
    public class WeaponSystemComponent
    {
        public event Action OnAttackStarted;
        public event Action OnAttackEnded;

        public EWeaponType CurWeaponType => curWeaponType;
        public bool IsAttacking => curAttack?.IsAttacking ?? false;

        private FieldCharacter owner;
        private PlayerAttributeSet attrSet;
        private EWeaponType curWeaponType;
        private BaseWeaponAttack curAttack;


        public void InitWSC(FieldCharacter _owner, PlayerAttributeSet _attrSet)
        {
            owner = _owner;
            attrSet = _attrSet;
        }

        public void ChangeWeaponType(EWeaponType _weaponType)
        {
            if (curAttack != null)
            {
                curAttack.OnAttackStarted -= OnAttackStartedCallback;
                curAttack.OnAttackEnded -= OnAttackEndedCallback;
            }

            curWeaponType = _weaponType;

            string typeName = $"MS.Battle.{_weaponType}Attack";
            Type attackType = Type.GetType(typeName);
            if (attackType == null)
            {
                Debug.LogError($"[WSC] 무기 공격 클래스를 찾을 수 없음: {typeName}");
                curAttack = null;
                return;
            }

            curAttack = Activator.CreateInstance(attackType) as BaseWeaponAttack;
            if (curAttack == null)
            {
                Debug.LogError($"[WSC] 무기 공격 인스턴스 생성 실패: {typeName}");
                return;
            }

            curAttack.InitWeaponAttack(owner, attrSet);
            curAttack.OnAttackStarted += OnAttackStartedCallback;
            curAttack.OnAttackEnded += OnAttackEndedCallback;
        }

        public void ActivateAttack() => curAttack?.OnAttackInput();

        private void OnAttackStartedCallback() => OnAttackStarted?.Invoke();
        private void OnAttackEndedCallback() => OnAttackEnded?.Invoke();
    }
}
```

### 특이사항

1. **`SetScaleX` 방향 컨벤션 변경 반영**: 작업 도중 사용자가 플레이어 캐릭터의 디폴트 facing 방향을 왼쪽→오른쪽으로 뒤집는 수정을 했고, `SetScaleX(_right ? 1f : -1f)`로 변경됨. 따라서 `IsFacingRight => Skeleton.ScaleX > 0f`가 올바른 방향(스펙 작성 당시 메모와 부호 반대). 구현 단계에서 실제 코드를 확인하여 정정함.

2. **`DamageInfo`는 struct + public fields 구조**: `DamageInfo.cs`에 명시적 생성자가 정의되어 있지 않아 **object initializer 구문**(`new DamageInfo { ... }`)으로만 생성 가능. 레퍼런스 코드(`MortalSoulDoc/04. Assets/02. Scripts/Skill/StatusEffect/StatusEffectUtils.cs`)는 named args 생성자 호출 패턴을 사용하지만 그 코드는 이전 프로젝트 잔재라 현재 구조와 다름.

3. **속성 enum 이름**: 스펙 본문에서는 무속성을 "Void"로 표기했으나 실제 enum 이름은 `EDamageAttributeType.None` (`= 0`). `Void`는 enum에 존재하지 않음. 코드에서는 `None`을 사용.

4. **무기 공격 클래스 리플렉션 탐색 — 어셈블리 동일 가정**: `Type.GetType("MS.Battle.{WeaponType}Attack")`는 호출 어셈블리 + mscorlib만 검색한다. WSC와 모든 `*Attack` 클래스가 `Assembly-CSharp` 동일 어셈블리에 있으므로 정상 동작. 향후 `WeaponAttack/` 폴더를 별도 Assembly Definition으로 분리할 경우 fully-qualified name(`"MS.Battle.OneHandSwordAttack, AssemblyName"`)으로 전환 필요.

5. **인게임 검증 도중 발견한 비-리팩터 이슈**: 런타임 검증 중 플레이어가 공중 정지/입력 무반응 상태로 시작되는 문제 발견. 진단 결과 `PlayerCharacter.prefab`의 `Rigidbody2D.bodyType`이 `Kinematic`이고 PlayerInput 컴포넌트의 Behavior가 Send Messages가 아니어서 발생한 **프리팹 설정 이슈**로, 본 리팩터 범위 외. 사용자가 프리팹을 수정하여 해결.

6. **공격 종료 후 캐릭터 동결 — Spine 이벤트 미부착이 원인**: OneHand 공격 흐름은 `WaitForAnimEventAsync("attack")` → 히트판정 → `WhenAny(combo_ready, Complete)` → 종료 흐름으로 짜여 있으나, 현재 Spine 데이터에 `attack`/`combo_ready` 이벤트가 아직 부착되어 있지 않아 첫 번째 await에서 영원히 대기. 애니메이션은 끝까지 재생되고 마지막 프레임 정지, `finally` 블록의 `InvokeAttackEnded()` 호출 안 됨 → 상태머신이 Attack 상태에 고정. **본 리팩터의 결함이 아니라 Spine 이벤트 부착 후속 작업의 선행 의존**. 사용자가 후속 작업으로 이벤트를 부착하면 정상 흐름 완성 예정.

7. **점프 작동 안 함 — 지면 판정 이슈 발견**: 인게임 검증 중 점프 입력은 들어오지만 `isGrounded=false`로 점프 분기에서 막히는 현상. `UpdateGroundCheck`의 BoxCast가 지면을 못 잡는 것으로 추정. 본 리팩터 범위 외(이동/점프/대시 시스템 자체는 건드리지 않음). 별도 작업으로 처리 예정.

8. **임시 진단 로그 정리**: 작업 도중 `PlayerCharacter.InitTestAsync`와 `PlayerMovementController.InitController`/`OnFixedUpdate`/`OnMove`/`OnJump`에 임시 Debug.Log를 추가하여 런타임 흐름을 추적했다. 작업 마무리 시 모두 제거함.
