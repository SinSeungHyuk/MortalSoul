### 작업 요청

마을 NPC 스토리 진행 및 튜토리얼에 사용할 **다이얼로그(대화) 시스템 1단계**를 구현한다.
데이터는 JSON SettingData, UI는 팝업 1종, 호출부는 UniTask 기반. NPC 필드 오브젝트 본구현은 별도 작업.

---

### 설계 요약 (브레인스토밍 결과 반영)

- **라인 데이터 구조**: 화자명 + 포트레이트 스파인키 + 스킨키 배열 + 좌/우 플래그 + 텍스트
- **저장 방식**: JSON `DialogueSettingData` (Dictionary, DataManager 로드)
- **타이핑 효과**: TMP `maxVisibleCharacters` + 2단계 터치 (타이핑 중→즉시완성 / 완성후→다음라인)
- **타이핑 속도**: `Settings.DialogueTypingSpeed`
- **아키텍처**: 전용 매니저 없이 `UIManager.ShowDialogueAsync(key)` + `DialoguePopup`가 로직 소유
- **호출 패턴**: `UniTask` 반환 → `await` / `Forget()` 선택
- **포트레이트**: `SkeletonGraphic` 사용, 스킨 배열을 조합(Combined Skin) 적용 — 에셋 특성상 여러 스킨키 합성 필수
- **입력 정책**: 팝업이 화면 전체를 덮는 단일 투명 버튼(백드롭)으로 터치를 독점. 외부 UI 비활성화는 하지 않음

### 1. IInteractable async 전환 (선행 수정)

상호작용 후 자연스러운 대화 체이닝(예: 게이트키퍼 → 대화 → 던전 진입)을 위해 시그니처를 비동기로 전환한다.

```csharp
public interface IInteractable
{
    string InteractIconKey { get; }
    UniTask InteractAsync(PlayerCharacter _player);
}
```

- 동기 구현체는 `return UniTask.CompletedTask;` 한 줄
- `PlayerInteractController.TryInteract()` → `CurTarget?.InteractAsync(Main.Instance.Player).Forget();`
- 기존 InProgress 문서(`상호작용 시스템 구현.md`)의 시그니처 블록도 이 결정 반영

### 4. DialoguePopup (신규)

`Assets/02. Scripts/UI/DialoguePopup.cs`

Awake에 자식 참조 캐시(`FindChildDeep`), `[SerializeField]` 미사용:

- `Button backdropButton` — 화면 전체 덮는 투명 버튼 (터치 수집)
- `TMP_Text speakerText`, `contentText`
- `SkeletonGraphic portraitLeft`, `portraitRight` — 사용 안 하는 쪽은 비활성

상태: `bool isTyping`, `UniTaskCompletionSource touchTcs`

API:
```csharp
public async UniTask PlayDialogueAsync(string _dialogueKey)
{
    var data = Main.Instance.DataManager.SettingData.DialogueDict[_dialogueKey];
    foreach (var line in data.Lines)
    {
        ApplyPortrait(line);
        speakerText.text = line.SpeakerName;
        await TypeTextAsync(line.Text);
        await WaitForTouchAsync();
    }
}
```

구현 포인트:
- **SkeletonDataAsset 교체**: Addressable로 로드 → `skeletonGraphic.skeletonDataAsset = asset; skeletonGraphic.Initialize(true);`. 같은 키 연속 사용 시 재초기화 스킵
- **스킨 조합**: `skeleton.Data.FindSkin(key)` 순회하여 빈 Skin에 `AddSkin` → `SetSkin(combined)` + `SetSlotsToSetupPose()`
- **타이핑**: `maxVisibleCharacters` 0→Length, 터치 시 `isTyping=false`로 빠져나와 즉시 Length
- **2단계 터치**: 백드롭 onClick 단일 핸들러에서 `isTyping` 분기, 둘 다 `touchTcs.TrySetResult()`
- **정리**: `PlayAsync` 완료 후 UIManager가 Destroy

### 5. UIManager 확장

```csharp
public async UniTask ShowDialogueAsync(string _dialogueKey)
{
    var prefab = await Main.Instance.AddressableManager.LoadResourceAsync<GameObject>("DialoguePopup");
    var go = GameObject.Instantiate(prefab, PopupCanvas);
    var popup = go.GetComponent<DialoguePopup>();
    try { await popup.PlayAsync(_dialogueKey); }
    finally { GameObject.Destroy(go); }
}
```

1차 구현은 동시 호출 금지(호출부 책임). 내부 가드는 차후.

### 특이사항 / 메모

- **SkeletonGraphic Multiple CanvasRenderers**: 포트레이트가 여러 텍스처 스킨 조합을 쓰므로 `allowMultipleCanvasRenderers = true` 필수. 안 켜면 경고 + 일부 슬롯 미렌더
- **타이핑 속도**: 상수 미사용(과한 추상화 회피), `1f / 30f` 직접 인라인
- **레이아웃 임시값**: 포트레이트 300x400, localScale 3배 (좌 `[3,3,1]` / 우 `[-3,3,1]` 플립), anchoredPosition ±280·하단 280. 실제 디자인 투입 시 재조정 필요
- **플레이어 대사 시 현재 장착 소울 스킨 반영**: 이번 스펙 외. 추후 `SoulGameData.MainSoulKey` 참조해서 JSON `SkinKeyList` 대신 런타임 주입으로 전환 예정
- **프리팹 로드 hitch**: 첫 대화 호출 시 Addressable 로드 + SkeletonGraphic Initialize hitch 가능. 프리로딩 전략은 추후
- **동시 대화**: 1차 구현은 호출부 책임. 가드 미구현
