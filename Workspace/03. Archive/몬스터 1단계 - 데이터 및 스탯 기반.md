### 작업 요청

몬스터 구현의 데이터 토대를 마련한다. MonsterAttributeSet/MonsterSettingData를 설계 문서에 맞게 갱신하고, BSC에 사망 가드를 추가한다.
이 작업은 이후 몬스터 2단계(MonsterCharacter/Controller), 3단계(MonsterManager/스폰)의 전제 조건이다.

> 참조: `MortalSoulDoc/04. Assets/Monster 설계.md` (§1, §3, §7-3, §9)

---

### 1. MonsterAttributeSet에 DetectionRange 추가

`Assets/02. Scripts/Battle/AttributeSet/MonsterAttributeSet.cs`

- `DetectionRange` 스탯 추가 (`AttackRange`와 동일 패턴)
- `InitAttributeSet`에서 `_data.DetectionRange` 값으로 초기화
- `statDict`에 `EStatType.DetectionRange` 등록

`Assets/02. Scripts/Battle/Stat.cs`의 `EStatType`에 `DetectionRange` 항목 추가.

---

### 2. MonsterAttributeSetSettingData에 DetectionRange 필드 추가

`Assets/02. Scripts/Data/SettingData/MonsterSettingData.cs`

- `MonsterAttributeSetSettingData` 클래스에 `public float DetectionRange { get; set; }` 추가

---

### 3. MonsterSkillSettingData 슬림화

`Assets/02. Scripts/Data/SettingData/MonsterSettingData.cs`

설계 문서 §7-3에 따라 레거시 필드 제거:
- `AnimTriggerKey` 제거 (스킬 구현체 내부에서 애니메이션 직접 제어)
- `SkillDuration` 제거 (await 기반 자연 종료)

유지 필드:
- `SkillKey` (스킬 식별 키)
- `SkillActivateRate` (Rate 기반 랜덤 선택 가중치)

---

### 4. BSC TakeDamage 사망 가드 추가

`Assets/02. Scripts/Battle/BattleSystemComponent.cs`

설계 문서 §9에 따라:
- `TakeDamage` 진입부에서 `owner.ObjectLifeState != FieldObjectLifeState.Live`이면 즉시 return
- BSC가 `owner` 참조를 보유해야 하므로, `InitBSC`에서 `_owner`를 필드에 저장

---

### 5. MonsterSettingData.json 갱신

`Assets/04. Settings/SettingData/MonsterSettingData.json`

- 기존 몬스터 데이터에 `DetectionRange` 필드 추가 (테스트용 기본값)
- `AnimTriggerKey`, `SkillDuration` 필드 제거

---

없음 (첫 번째 작업)

- 몬스터 2단계: MonsterCharacter + MonsterController 구현

### 작업 내용
- MonsterSettingData 슬림화 완료 (AnimTriggerKey, SkillDuration 제거)
- BSC TakeDamage 사망 가드 추가 (Health <= 0 시 return)
- DetectionRange는 MonsterAttributeSet 스탯 대신 Settings.MonsterDetectionRange 상수로 대체 구현

### 특이사항
- DetectionRange를 per-monster 스탯이 아닌 전역 상수로 처리. 추후 몬스터별 탐지 거리가 필요하면 MonsterAttributeSet으로 이동 필요
