### 작업 요청

NPC 대화 / 던전 포탈 입장 / 상점 등 게임 전반에 사용될 **상호작용 시스템**을 구현한다.
HUD 고정 위치의 상호작용 버튼과 트리거 기반의 단일 타겟 슬롯 구조로, 추후 확장(상자, 제단, 이벤트 오브젝트 등)을 염두에 둔다.

---

### 설계 요약

- **HUD 고정 버튼**: 조이스틱/공격버튼처럼 화면에 항상 떠있음
- **단일 타겟 슬롯**: PlayerInteractor가 현재 타겟 1개만 보유 (Enter 시 덮어쓰기, Exit 시 무조건 null)
- **인터페이스 기반**: `IInteractable` 구현체는 자유롭게 확장
- **아이콘은 키 기반**: SkillSettingData.IconKey 컨벤션 따라 Addressable Sprite 키로 노출

> 트리거 중첩에 대한 후보 관리/우선순위 로직은 의도적으로 제외 (오버엔지니어링 회피, 기획 단계에서 회피)

### 2. PlayerInteractor 신규 작성

`Assets/02. Scripts/FieldObject/FieldCharacter/Player/PlayerInteractor.cs`

- `MonoBehaviour` 상속
- 플레이어 자식 GameObject `InteractTrigger`에 부착
  - `BoxCollider2D` (IsTrigger = true)
  - `Rigidbody2D` (BodyType = Kinematic)
  - Layer = `Interactable`만 충돌하도록 설정
- 필드/이벤트:
  - `IInteractable CurTarget { get; private set; }`
  - `event Action<IInteractable> OnTargetChanged`
- 메서드:
  - `OnTriggerEnter2D` → `IInteractable` 캐스팅 성공 시 CurTarget 대입 + 이벤트 발행
  - `OnTriggerExit2D` → CurTarget = null + 이벤트 발행 (식별 비교 없이 단순 null 처리)
  - `TryInteract()` → `CurTarget?.Interact(Main.Instance.Player)`

### 4. HUD 상호작용 버튼 구현

`Assets/02. Scripts/UI/HUD/` (HUD 뷰 스크립트 위치 — 기존 HUD 구조에 맞춰 배치)

- 버튼 클릭 핸들러: `OnBtnInteractClicked()` → `Main.Instance.Player.Interactor.TryInteract()`

### 특이사항

- **컴포넌트 분리 결정**: PlayerInteractController를 별도 MonoBehaviour로 분리. 이유는 트리거 콜라이더가 레이어 분리상 자식 GameObject에 위치해야 하므로 `OnTriggerEnter2D` 수신 주체가 자식에 있어야 함. 기존 PMC/PSC/BSC 책임 분리 컨벤션과도 일치.
- **Player Layer 재사용**: InteractTrigger Layer를 별도 분리 없이 Player로 유지. 현재 `BattleObject.IsValidTarget`이 `TryGetComponent<FieldCharacter>` 체크를 통과해야만 피격 처리하므로 InteractTrigger(FieldCharacter 없음)는 자동 제외 → 중복 피격 위험 없음. 단, 향후 몬스터 스킬에서 `Physics2D.OverlapXxx(..., PlayerLayer)` 사용 시 FieldCharacter 체크 컨벤션 강제 필요.
- **테스트 버튼은 TestScene의 루트 Canvas(TestUIController 부착)에 배치**. 본 HUD 구조가 확정되면 `Main.UI.ViewCanvas`로 이동 예정.
- **작업요청 원문의 `PlayerInteractor` 명칭은 `PlayerInteractController`로 확정** (PMC/PSC와 컨벤션 통일).
