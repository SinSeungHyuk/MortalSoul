# 게임 진행 흐름 메모

> 브레인스토밍으로 확정한 사항 누적. (최종 갱신: 2026-04-14)

---

## 1. 영속성 모델

- **완전 런 기반**. 영혼/정수/경험치/레벨 모두 한 런 한정
- 런 종료 시 전부 소실
- 영구 성장(메타 프로그레션)은 추후 별도 스펙
- 설정값(사운드 등)만 영구 저장 — 기존 SoundManager가 처리 중

---

## 2. 전체 루프

```
타이틀(터치) → 초기 로딩 → 마을 → NPC 대화 → 던전 진입
  → 방 클리어 반복 → 보스 or 사망 → 결과 팝업 → 마을 자동 복귀
```

---

## 3. 마을 (미니멀)

- NPC 1명만 배치
- NPC = 대화 시스템만 보유. 던전 입장은 NPC 대화 선택지로 진행
- 전투/허수아비 없음
- 복귀 시 고정 스폰, HP 풀피, 기본 메인 소울 1개, 서브 슬롯 빔

---

## 4. 던전 구조 (세피리아식 7레이어)

| 레이어 | 역할 | 노드 수 |
|--------|------|---------|
| 1 | 마을 | 1 고정 (1-1) |
| 2~6 | 던전 구간 | 각 2~3 랜덤 |
| 7 | 보스 | 1 고정 (7-1) |

- 좌표 표기: `n-m` (n=레이어, m=위치)
- **레이어 간 이동 제약 없음** — 다음 레이어의 모든 노드 자유 선택

### 노드 타입 배치 (쿼터 기반)

- 타입: 일반 전투 / 엘리트 / 상점 / 이벤트
- **최소 쿼터 보장**:
  - 엘리트: min 1 / max 1
  - 상점: min 1 / max 2
  - 이벤트: min 2 / max 3
  - 나머지는 일반 전투
- 소프트 제약: 엘리트는 5층부터** 등장, 2층은 무조건 모두 일반전투
- 생성 규칙/가중치는 JSON으로 관리

### 던전 맵 UI

- 좌우 스크롤(ScrollRect)
- 노드 아이콘 + 화살표 연결
- 방문가능/완료/미방문 상태 시각 구분

---

## 5. 방 공통 규칙

- **원씬 구조** — 씬은 던전 하나. 방은 프리팹 스왑으로 교체
- **모든 방은 물리적 조작 가능 공간**. 텍스트 전용 방 없음
- 기본 방 크기: 카메라 **2개 길이** 좌우 스크롤 아레나
- 방 클리어 조건: 모든 몬스터 처치 → 포탈 등장
- **포탈은 상호작용 시스템 재사용** (NPC/오브젝트와 공용). 근접 시 버튼 표시, 터치로 진입

### GameManager 상태 세분화 (구현 방향)

```
EGameState
├─ Title
├─ Village
└─ Dungeon_*
   ├─ Dungeon_Map
   ├─ Dungeon_Battle
   ├─ Dungeon_Elite
   ├─ Dungeon_Shop
   ├─ Dungeon_Event
   └─ Dungeon_Boss
```

각 상태 Enter에서 방 로드 + 스폰 + HUD/BGM 스왑 전담.

---

## 6. 방 타입별

### 일반 전투방 (Dungeon_Battle)
- 몬스터 **일괄 스폰** (웨이브 X)
- 레이어별 몬스터 수/종류 스케일링 (JSON)
- 전원 처치 → 포탈

### 엘리트방 (Dungeon_Elite)
- **사전 제작 중간보스 1체 확정 스폰** (키값 기반)
- 1:1 구도, 호위 없음
- 전용 BGM, 상단 보스 체력바 HUD

### 이벤트방 (Dungeon_Event)
- 물리적 공간. 이벤트 전용 오브젝트/NPC 배치
- 플레이어가 상호작용 시 대화 시스템 → 선택지 → 효과 (미정)
- **세부 이벤트 풀/구현은 추후**

### 상점방 (Dungeon_Shop)
- 물리적 공간. 상점 NPC 배치
- 재화: 영혼의 정수
- 품목 방향: 영혼 1~2 + 스탯 업그레이드 2~3 + 체력 회복 1
- **가격/리롤/품목 세부는 추후**

### 보스방 (Dungeon_Boss, 7-1)
- 사전 제작 보스 1체 확정 스폰
- **다단계 페이즈** 보스
- 페이즈 전환 로직은 **보스 전용 상태머신**에서 처리 (추후, 몬스터별 상태머신 분화 계획과 연계)
- 전용 BGM, 입장 연출, 약간 넓은 방
- 처치 = 런 클리어, 보상 없이 즉시 종료 단계

---

## 7. 방 클리어 보상

- **경험치**: 몬스터 처치 시 UI 경험치바 즉시 반영 (드랍 오브 없음)
- **영혼의 정수**: 몬스터 처치 시 필드 아이템으로 드랍 → 트리거 없이 **자동 흡수 연출**로 수령
- **영혼(Soul) 드랍 규칙**:
  | 방 | 드랍 |
  |---|---|
  | 일반 전투방 | 전 등급 랜덤 풀에서 1개 확정 |
  | 엘리트방 | 상위 등급 풀에서 1개 확정 |
  | 이벤트/상점/보스 | 드랍 없음 |
  - 전투 계열 방은 100% 영혼 1개 보장
  - 영혼은 드랍 아이템 아닌 **팝업 UI**로 제공 (아래 시퀀스)

---

## 8. 방 클리어 시퀀스 (핵심)

```
마지막 몬스터 처치
  ↓
영혼 획득 팝업 (슬롯 꽉 차면 교체/버리기 UI)
  ↓ 팝업 종료
포탈 등장 (플레이어 자유시간)
  ↓ 포탈 상호작용
레벨업 보상 팝업 (누적분만큼 연속)
  ↓ 선택 완료
던전 맵 UI 오픈 → 다음 노드 선택
```

**순서 의도**: 영혼 타입을 먼저 확인 → 레벨업 스탯 선택에 반영 (영혼 획득 전에 스탯 올리면 손해 방지).

---

## 9. 레벨업 보상 (최소 구현)

- **3택 고정**
- **스탯 증가만** (특수 카드 없음)
- **리롤 없음**
- 카드 등급 시스템 (Common/Rare/Unique/Legendary) — JSON에 등급별 스탯 증가량 + 가중치 정의
- 추첨: 등급 가중치로 등급 결정 → 해당 등급 풀에서 스탯 추첨
- 중복 방지 (한 팝업 내 3장 스탯 안 겹침)
- `AttackRange`는 풀에서 제외 (몬스터 전용)
- 다회 레벨업 시 독립 3택 연속 팝업

---

## 10. 런 종료

- **보스 클리어**: 즉시 VICTORY 연출 → 결과 요약 팝업 → "마을로" 버튼 → 자동 복귀
- **플레이어 사망**: 즉시 DEFEATED → 결과 팝업 → "마을로" → 자동 복귀
- **부활/리트라이 없음**
- 종료 시 정리: GameData 해제, MonsterManager/BattleObjectManager/EffectManager 클리어, SSC 스킬 캔슬, 플레이어 리셋, HUD 숨김, 상태 Village 전환

---

## 11. 대화 시스템 (표준)

- 하단 텍스트 + 좌/우 포트레이트 + 선택지 버튼
- **포트레이트는 Spine UI 렌더링**(SkeletonGraphic) 사용 → 스킨 키로 관리
- 데이터 구조(JSON): DialogKey / Lines(Speaker, Text, PortraitSkinKey, Side) / Choices(Text, EffectKey, NextDialogKey)
- 호출: `UIManager.ShowDialog(key, onComplete)`
- 적용처: 마을 NPC / 이벤트 오브젝트·NPC / 상점 NPC

---

## 12. 상호작용 시스템 (공용)

- 대상: NPC, 포탈, 상점 주인, 이벤트 오브젝트 등
- 근접 시 상호작용 버튼 표시 → 터치 시 대상별 콜백
- 단일 범용 시스템 — 재사용성 어필

---

## 13. 모바일 처리

- 앱 전환: `OnApplicationPause` → 자동 일시정지 + "계속하기" 수동 재개
- 일시정지 메뉴: 계속 / 설정 / 마을 귀환(런 포기) / 타이틀
- **저장: 런 자체 휘발** (중단 이어하기 없음). 설정값만 영구 저장
- **가로 고정** (Landscape), CanvasScaler 기반 해상도 대응
