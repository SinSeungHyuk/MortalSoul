# 게임 진행 흐름 메모

> 브레인스토밍 진행 중. 확정된 사항을 누적 정리.

---

## 1. 타이틀 → 마을

- 타이틀 UI → 화면 터치 → 마을 진입
- 마을 진입 시 마을 맵 프리팹 로드 + 캐릭터 스폰 + 조작 시작

---

## 2. 마을 구성

- NPC 1명 배치 (추후 개발 진도에 따라 NPC/컨텐츠 확장)
- NPC는 초기에 다이얼로그 대화 시스템만 보유
- 던전 입장은 별도 입구 오브젝트 없이 **NPC 대화를 통해 진행**
  - 스토리 설명 가능
  - 포트폴리오에서 대화 시스템 어필
  - 자연스러운 입장 흐름

---

## 3. 상호작용 시스템 (공용)

- 플레이어가 NPC 근접 시 **상호작용 버튼** 표시 → 터치로 대화 시작
- 이 시스템은 NPC, 상점, 오브젝트 등에 **재사용 가능하도록 범용 설계**
- 구체적 구현 방식(트리거 콜라이더, UI 버튼 위치 등)은 구현 단계에서 확정

---

## 4. 던전 맵 UI

- 던전 진입 후 **전체 던전 구조를 월드맵처럼 UI로 표시**
- **좌우 스크롤**(ScrollRect) 방식으로 전체 경로 탐색
- 각 노드(전투/상점/이벤트/보스 등)를 아이콘으로 배치
- 노드 간 연결을 **화살표 이미지**로 표시
- 슬레이 더 스파이어 스타일의 세로 레이어 기반 배치 (레이어당 1~3개 노드)
- 구체적인 던전 생성 규칙(레이어 수, 분기/합류 규칙 등)은 추후 확정

---

## 미확정 항목

- [ ] 던전 맵 생성 규칙 (레이어 수, 노드 수, 분기/합류)
- [ ] 던전 내부 진행 흐름 (전투 → 보상 → 맵 선택 루프)
- [ ] 방 클리어 후 보상 방식
- [ ] 던전 클리어/실패 조건 및 마을 복귀 흐름
- [ ] 대화 시스템 상세 (분기 선택지, 스크립트 데이터 구조 등)
